# Observability — Prometheus & Grafana

The backend exposes Prometheus metrics at **`GET /metrics`** (unauthenticated by
design — restrict at the network). An optional Compose **observability profile**
wires up Prometheus (scrape + store) and Grafana (a pre-provisioned dashboard) so
the metrics become visible with zero manual setup.

## Start it

```bash
docker compose --profile observability up -d prometheus grafana
```

- **Prometheus** → <http://localhost:9090> (scrapes `backend:8000/metrics` every 15s)
- **Grafana** → <http://localhost:3000> (login `admin` / `admin` by default — override
  with `GRAFANA_ADMIN_USER` / `GRAFANA_ADMIN_PASSWORD`)

Grafana auto-provisions the Prometheus datasource and the **"SentinelAI — Overview"**
dashboard (folder *SentinelAI*). No manual import needed.

```
infra/
├── prometheus/prometheus.yml                       # scrape config
└── grafana/
    ├── provisioning/datasources/prometheus.yml     # Prometheus datasource
    ├── provisioning/dashboards/dashboards.yml       # dashboard provider
    └── dashboards/sentinelai-overview.json          # the dashboard
```

## Dashboard panels

Built on the metrics in `app/core/metrics.py`:

| Panel | Metric(s) |
| --- | --- |
| Active WebSocket clients | `sentinelai_websocket_active_connections` |
| Alerts created (24h) | `sentinelai_detection_alerts_total` |
| Events classified (24h) | `sentinelai_detection_events_total` |
| Model drift score | `sentinelai_drift_score` |
| HTTP request rate by status | `sentinelai_http_requests_total` |
| HTTP p95 latency by route | `sentinelai_http_request_duration_seconds_bucket` |
| Detection pipeline (runs/events/alerts) | `sentinelai_detection_*_total` |
| Response actions by status | `sentinelai_response_actions_total` |
| Ingestion rows by outcome | `sentinelai_ingestion_rows_total` |
| Reports generated by kind | `sentinelai_reports_total` |

## On Kubernetes

Backend pods carry `prometheus.io/scrape` / `prometheus.io/port` annotations, so a
cluster-wide Prometheus (e.g. kube-prometheus-stack) discovers `/metrics`
automatically. Import `infra/grafana/dashboards/sentinelai-overview.json` into
your existing Grafana. See [infra/k8s/README.md](../infra/k8s/README.md).
